# 智能园徽 - 产品文档

> 本文档记录「智能园徽」小程序的产品定义、页面结构及交互逻辑。

---

## 一、产品概述

### 1.1 产品定位
「智能园徽」是一款面向幼儿园场景的设备管理小程序，用于管理学生佩戴的智能园徽设备，支持设备状态查看、活动管理及学生报告导出等功能。

### 1.2 目标用户
- 幼儿园管理员
- 班主任/教师

### 1.3 核心功能
- **设备管理**：查看设备列表、状态监控、添加/编辑/删除设备
- **设备名册**：设备信息总览，查看网关与园徽的归属关系
- **活动列表**：园区活动创建、编辑、删除与详情查看
- **学生报告**：学生活动数据报告（待实现）

- **功能按钮区**：4 个按钮纵向排列，各占满整行
- **使用教程区**：位于按钮下方，展示图文教程卡片

#### 功能按钮

|------|------|------|
| 设备管理 | 蓝色 | 进入设备管理页 |
| 设备名册 | 绿色 | 进入设备名册页 |
| 活动列表 | 橙色 | 进入活动列表页 |

#### 使用教程
- 纵向排列的图文卡片
- 每张卡片包含：占位图区域 + 步骤说明文字

### 2.2 设备管理

#### 布局
- **顶部导航栏**：左侧返回箭头 + 标题「设备管理」
- **设备列表区**：纵向排列的设备卡片
- **底部固定栏**：「添加设备」主按钮，吸底展示

#### 设备卡片
| 字段 | 说明 |
|------|------|
| 设备名称 | 如：网关-001 |
| 设备状态 | 用颜色圆点表示，绿色为正常，灰色为离线 |

#### 底部操作抽屉
点击设备卡片后，从底部弹出操作抽屉：

| 操作 | 说明 |
|------|------|
| 详情 | 进入设备详情页 |
| 编辑 | 打开「编辑设备」弹窗 |
| 删除 | 打开「删除确认」弹窗 |
| 取消 | 关闭抽屉 |

##### 编辑设备弹窗
- 弹窗标题：「编辑设备」
- 表单字段：
  - 设备名称（文本输入，必填，默认填充当前名称）
  - 设备 MAC（文本输入，选填，默认填充当前 MAC）
- 底部按钮：「取消」（不保存）、「确定」（校验名称非空后保存，刷新列表）

##### 删除确认弹窗
- 弹窗标题：「请确认是否删除网关设备」
- 提示文案：「删除后不可恢复，请谨慎操作」
- 底部按钮：「取消」（终止删除）、「确认」（红色按钮，删除后刷新列表）

##### 添加设备弹窗
点击底部「添加设备」按钮后弹出：
- 弹窗标题：「添加设备」
- 表单字段：
  - 设备名称（文本输入，必填）
  - 设备 MAC（文本输入，选填）
- 底部按钮：「取消」（不保存）、「确定」（校验名称非空后保存，刷新列表）

### 2.3 设备名册

#### 布局
- **顶部导航栏**：左侧返回箭头 + 标题「设备名册」
- **班级筛选栏**：导航栏下方，横向平铺展示所有班级标签
- **当前活动卡片**（筛选项下方）：展示目标班级当前进行中的活动信息
- **学生卡片列表区**：当前活动卡片下方，纵向排列的学生与园徽对应关系卡片

#### 班级筛选
- **标签排列**：横向平铺展示所有班级标签，支持左右滑动
- **标签选项**：大班-1班、大班-2班、中班-1班、中班-2班、小班-1班、小班-2班（无「全部」选项）
- **选中态**：绿色背景 + 白色文字；未选中态：白色背景 + 灰色文字 + 灰色边框
- **点击切换**：点击后高亮选中，下方内容实时刷新
- **默认选中**：首次进入默认选中用户拥有的第一个班级（由后端权限控制）

#### 当前活动卡片
位于班级筛选栏下方，独立展示的通栏卡片：

| 字段 | 说明 |
|------|------|
| 活动名称 | 如：画一匹奔腾的小野马 |
| 开始时间 | 如：2026/06/01 09:00 |
| 结束时间 | 如：2026/06/01 09:40（开始时间 + 活动时长）|
| 状态标签 | 未开始（灰色）/ 进行中（绿色）|

- **展示规则**：仅展示该班级下状态为「未开始」或「进行中」的活动，一个班级同一时间最多只有一个符合条件的活动
- **点击交互**：点击卡片跳转至该活动的活动详情页
- **视觉风格**：白色圆角卡片，左侧绿色竖条装饰，带阴影

#### 空状态
当目标班级当前没有「未开始」或「进行中」的活动时：
- **不展示**当前活动卡片
- **不展示**学生与设备名册列表
- 页面仅展示**缺省图**：居中占位图标 + 「该班级当前暂无进行中的活动」提示文案

#### 学生卡片
有当前活动时，在活动卡片下方展示学生卡片列表。每张卡片横向布局：

| 元素 | 说明 |
|------|------|
| 学生头像 | 左侧圆形默认头像占位图 |
| 学生姓名 | 如：王小明 |
| 园徽信息 | 右侧展示绑定的园徽编号，如「园徽-001」；未绑定则显示「未绑定」灰色文字 |

##### 卡片视觉
- **已绑定**：白色背景，园徽信息为绿色文字
- **未绑定**：白色背景，园徽信息为灰色文字「未绑定」

#### 数据对应规则
- 学生按班级归属分组，每个学生固定属于一个班级
- 园徽通过「持有人」字段与学生建立绑定关系
- 一个学生最多绑定一枚园徽，一枚园徽最多绑定一个学生
- **名册展示前提**：班级必须存在「未开始」或「进行中」的活动，否则不展示任何名册数据

### 2.4 设备详情

#### 布局
- **顶部导航栏**：左侧返回箭头 + 标题「设备详情」
- **内容区**：纵向排列的信息卡片 + 园徽列表
- **底部固定栏**：「返回」与「刷新数据」双按钮并排

#### 基本信息
白色圆角卡片，纵向展示 3 行信息：

| 字段 | 说明 |
|------|------|
| 设备名称 | 如：网关-001 |
| 设备 MAC | 如：A4:B5:C6:D7:E8:F9 |
| 设备状态 | 在线（绿色）/ 离线（灰色） |

#### 关联园徽列表
- **标题栏**：左侧绿色竖条 + 「关联园徽」+ 数量统计
- 纵向排列园徽卡片，每张卡片展示以下信息：

| 字段 | 说明 |
|------|------|
| 园徽编号 | 如：园徽-001 |
| 园徽 MAC | 如：A1:B2:C3:D4:E5:F6 |
| 园徽状态 | 正常（绿色）/ 禁用（灰色） |

##### 园徽操作抽屉

- **启用**：将该园徽状态恢复为「正常」
- **取消**：关闭抽屉

### 2.6 创建活动引导弹窗

#### 触发方式
点击活动列表页底部「+ 创建活动」按钮后弹出。

#### 目的
提醒用户在发布/编辑/删除活动前，必须先将所有徽章放置在充电箱内并连接电源，确保设备处于充电状态。

#### 布局
- **遮罩层**：半透明黑色背景，点击不可关闭
- **弹窗主体**：居中白色卡片
  - 引导图区：顶部占位图片（充电箱操作示意图）
  - 文字描述区：
    > 发布/编辑/删除活动前，需要做如下准备工作：
    > 第一步：将所有徽章放置在充电箱内的充电插槽处；
    > 第二步：连接充电箱电源
    > 保证所有徽章设备处于充电状态，才可以进行操作哦
  - 底部按钮区：居中「我已知晓」按钮

#### 交互
点击「我已知晓」后关闭弹窗，进入创建活动页。

### 2.7 创建活动

#### 布局
- **顶部导航栏**：左侧返回箭头 + 标题「创建活动」
- **步骤指示器**：导航栏下方，三步进度指示器（圆点+步骤名称）
- **内容区**：根据当前步骤展示不同的表单内容
- **底部操作栏**：固定吸底，【上一步】【下一步】双按钮

#### 步骤总览
| 步骤 | 标题 | 内容 |
|------|------|------|
| 步骤一 | 选择活动设备 | 使用设备 + 园徽状态 + 上传任务提示 |
| 步骤二 | 选择参与人员 | 活动班级 + 参与幼儿 + 数量校验提示 |
| 步骤三 | 填写活动信息 | 活动名称 + 活动类型 + 开始时间 + 活动时长 |

#### 步骤指示器
- 三个圆点横向排列，中间用细线连接
- **当前步骤**：琥珀色实心圆点 + 琥珀色步骤名称
- **已完成步骤**：绿色对勾图标 + 灰色步骤名称
- **未完成步骤**：灰色空心圆点 + 灰色步骤名称

#### 底部操作栏
固定悬浮在页面底部的双大按钮栏，占据全宽，左右严格并排：

| 步骤 | 左侧按钮 | 右侧按钮 |
|------|----------|----------|
| 步骤一 | 取消（白色背景 + 灰色边框，圆角大按钮） | 下一步（琥珀色主按钮，圆角大按钮） |
| 步骤二 | 上一步（白色背景 + 灰色边框，圆角大按钮） | 下一步（琥珀色主按钮，圆角大按钮） |
| 步骤三 | 上一步（白色背景 + 灰色边框，圆角大按钮） | 发布（琥珀色主按钮，圆角大按钮） |

- **布局规则**：两个按钮必须**相同大小**，使用 `flex: 1` 各占 50% 宽度，中间固定 12px 间隙
- **按钮尺寸**：高度不低于 44px，圆角 0.75rem，文字字号 0.9375rem，字重 500
- **悬浮固定**：`position: fixed; bottom: 0; left: 0; right: 0`，带半透明毛玻璃背景（`backdrop-filter: blur(8px)`），避免内容被底部栏盖住（内容区底部增加约 80px 的 padding 或占位）
- **步骤切换时内容区直接替换**，无过渡动画
- **左上角返回箭头点击后直接返回活动列表页**，不做步骤回退
- **步骤二的「下一步」按钮**：当可用园徽数量 < 参与幼儿数量时，按钮置灰（`opacity: 0.45; pointer-events: none`），不可点击

#### 步骤一：选择活动设备

##### 使用设备
- **布局**：横向平铺，一行4个紧凑型小卡片，宽度严格一致
- **内容**：仅展示设备名称（如：网关-001）
- **选中态**：琥珀色边框 + 浅琥珀色背景
- **未选中态**：白色背景 + 灰色边框
- 点击卡片即可选中，同一时间仅能选中一台设备
- **联动规则**：选中设备后，自动清空下方「园徽状态」及步骤二、步骤三的所有选择状态；同时加载并展示「园徽状态」区域

##### 园徽状态
选择设备后自动展开，标题栏包含「园徽状态」文案及右侧「刷新」按钮：

- **刷新按钮**：点击后模拟重新获取园徽最新状态，按钮带旋转动效反馈
- **布局**：网格布局，一行6个紧凑型卡片

每张园徽状态卡片为纵向紧凑卡片，从上到下分为三个区域：

| 区域 | 说明 |
|------|------|
| 上半部分 | 园徽序号，如：001（仅展示序号数字，省略「园徽-」前缀） |
| 中间部分 | 连接状态文字，「已连接」（绿色）/「断开」（灰色） |
| 底部电量条 | 矩形进度条，展示当前电量百分比，颜色规则：≥60% 绿色、20%-60% 黄色、≤20% 红色 |

- **点击交互**：点击卡片切换该园徽的启用/禁用状态
- **启用状态视觉**：
  - 启用：白色背景卡片
  - 禁用：30% 透明度的浅红色背景（`rgba(254, 226, 226, 0.3)`）+ 整体内容置灰
- **状态独立展示**：连接状态、电量百分比、启用状态三个维度同时展示在卡片上

##### 上传任务提示板块
位于园徽状态区域下方，**仅当所选设备下存在「正在上传录音数据」的园徽时展示**：

- **视觉风格**：浅黄色背景圆角卡片，左侧带警告图标
- **提示文案**：「当前有 N 个园徽正在上传录音数据，请等待上传完毕」
- **交互按钮**：「终止上传」灰色边框按钮
- **隐藏条件**：没有正在上传的园徽时，整个板块不展示

###### 终止上传二次确认弹窗
点击「终止上传」后弹出：

- **弹窗标题**：「确认终止上传」
- **提示文案**：「终止上传后，园徽已录制的音频数据将会丢失，是否确认终止？」
- **底部按钮**：
  - 「取消」：关闭弹窗，不做任何操作
  - 「确认终止」：红色按钮，执行终止操作
- **终止后**：关闭弹窗，刷新园徽状态列表，对应园徽的上传状态变为已终止

#### 步骤二：选择参与人员

##### 活动班级
- **布局**：横向平铺，一行4个紧凑型小卡片，样式与步骤一「使用设备」完全一致
- **内容**：仅展示班级名称（如：大班-1班）
- **选中态**：琥珀色边框 + 浅琥珀色背景
- **未选中态**：白色背景 + 灰色边框
- 点击卡片即可选中，同一时间仅能选中一个班级
- **联动规则**：选中班级后，自动加载并展示下方的「参与幼儿」区域；同时清空「参与幼儿」内已选状态并默认全部选中

##### 参与幼儿
选择班级后自动展开，标题栏为「参与幼儿」，右侧展示已选人数/总人数：

- **布局**：网格布局，一行6个紧凑型卡片
- **内容**：每张卡片仅展示幼儿姓名
- **选中态（参与）**：绿色边框 + 浅绿色背景，右上角带白色对勾图标
- **未选中态（不参与）**：白色背景 + 灰色边框，文字置灰
- **点击交互**：点击卡片切换该幼儿的参与/不参与状态
- **默认状态**：首次加载时，默认全部幼儿为「参与」状态（全部选中）

##### 数量校验提示
位于参与幼儿区域下方，**每次选中/取消幼儿后实时更新**：

- **展示信息**：
  - 可用园徽数量：当前所选设备下，状态为「启用」且「在线」的园徽数量
  - 参与幼儿数量：当前步骤二中已选中的幼儿数量
- **实时校验与拦截**：
  - 若可用园徽数量 < 参与幼儿数量：
    - 展示红色错误提示卡片：「可用园徽数量不足，请减少参与幼儿或检查园徽设备状态」
    - **「下一步」按钮置灰不可点击**，用户必须减少参与幼儿数量或返回步骤一检查园徽状态后才能继续
  - 若数量充足：提示隐藏，「下一步」按钮恢复正常
- **底部信息栏**：固定展示「可用园徽：X 个 / 参与幼儿：Y 人」

---

#### 步骤三：填写活动信息

##### 活动名称
- 文本输入框，上限20字，必填
- 提交时校验不可与已有活动重名

##### 活动类型
- 多选标签：美术、音乐、体育、语言、科学、手工
- 全部平铺展示，点击选中，再次点击反选
- 选填项

##### 开始时间
- 日期时间选择器，默认回填当前时间，精确到分钟，必填

##### 活动时长
- **布局**：多行横向平铺的矩形按钮，每行5个选项，共两行
- **选项**：10分钟、20分钟、30分钟、40分钟、50分钟、60分钟、70分钟、80分钟、90分钟
- **选中态**：琥珀色背景 + 白色文字
- **未选中态**：白色背景 + 灰色文字 + 浅灰色边框
- 点击即可选中，同一时间仅能选中一个时长

---

#### 校验规则
1. **步骤一**：必须选择一项使用设备
2. **步骤二**：
   - 必须选择一项活动班级
   - 参与幼儿至少选择1人
   - **实时数量校验**：可用园徽数量必须 ≥ 参与幼儿数量，否则阻断进入下一步
3. **步骤三**：
   - 活动名称：不能为空；不能超过20字；不能与已有活动名称重复
   - 开始时间：不能为空
   - 活动时长：必须选择一项
4. **设备活动状态校验**（步骤三点击发布时执行）：检查所选设备是否存在「未开始」或「进行中」状态的活动。若存在，终止提交流程，提示：「{设备名称}已有将要开始/正在进行的活动，无法创建。请删除未开始的活动/等待活动结束再进行创建。」
5. **设备时间重叠校验**（步骤三点击发布时执行）
6. 活动类型为选填项，不做校验

#### 提交方式
- 步骤三点击「发布」时执行全部校验
- 校验通过后进入「发布进度弹窗」，展示硬件下发进度

#### 发布进度弹窗
点击「发布」并校验通过后弹出，展示活动下发到硬件设备的实时进度。

##### 进度节点
| 节点 | 说明 |
|------|------|
| 检查设备在线状态 | 检查所选网关设备是否在线 |
| 检查园徽链接状态 | 检查该网关下关联的园徽是否正常连接 |
| 发布到园徽 | 展示实时进度：{已发布数量}/{需发布数量}，其中「需发布数量」为关联到学生的园徽总数 |

##### 节点状态
每个节点均具备以下三种状态之一：

| 状态 | 视觉表现 |
|------|----------|
| 进行中 | 节点标题旁展示加载动效，节点处于等待完成状态 |
| 完成 | 节点标题旁展示完成图标，文字置灰 |
| 失败 | 节点标题旁展示失败图标，文字标红 |

##### 发布完成
- 全部节点完成后，弹窗保持打开，展示所有节点为完成状态
- 底部按钮变为「返回」，点击后关闭弹窗并返回活动列表页
- 新创建的活动按开始时间倒序插入列表

##### 发布失败
- 任一节点失败后，进度停止，展示失败状态
- 底部按钮变为：「取消」（关闭弹窗，返回创建页）+ 「重试」（重新执行发布流程）

### 2.8 编辑活动

#### 布局
同创建活动页，标题改为「编辑活动」，同样采用三步向导式流程。

#### 表单回填
打开页面时自动回填当前活动的全部字段，并根据数据定位到对应步骤：
- **步骤一**：已选中的使用设备高亮展示，下方展开对应园徽状态网格
- **步骤二**：已选中的活动班级高亮展示，下方展开参与幼儿网格，默认选中该活动原有的参与学生
- **步骤三**：活动名称、活动类型（已选中的类型高亮展示）、开始时间、活动时长全部回填

#### 底部操作栏差异
编辑活动底部右侧按钮文案为「保存」（创建活动为「发布」），其余与创建活动完全一致。

#### 校验规则
同创建活动页，但以下三点特殊处理：
- **活动名称重复校验时需排除当前活动自身**（允许保持原名称不变）
- **设备时间重叠校验时需排除当前活动自身**（允许保持原设备不变，或更换设备后重新校验）
- **设备活动状态校验时需排除当前活动自身**

#### 提交方式
- 步骤三点击「保存」时执行全部校验
- 校验通过后更新活动数据并返回活动列表页
- 不进入发布进度弹窗（编辑不涉及硬件重新下发）

### 2.9 活动详情

#### 布局
- **顶部导航栏**：左侧返回箭头 + 标题「活动详情」
- **内容区**：纵向排列的信息卡片、活动报告 Banner、参与人员列表

#### 基本信息
白色圆角卡片，纵向展示 7 行信息：

| 字段 | 说明 |
|------|------|
| 活动名称 | 如：画一匹奔腾的小野马 |
| 活动类型 | 琥珀色标签展示 |
| 活动班级 | 如：总园/小一班 |
| 开始时间 | 如：2026/06/01 09:00 |
| 活动时长 | 如：40 分钟 |
| 使用设备 | 如：网关-001 |
| 活动状态 | 标签展示：未开始（灰色）、进行中（绿色）、已结束（蓝色） |

##### 活动状态计算规则
- **未开始**：当前时间 < 活动开始时间
- **进行中**：活动开始时间 ≤ 当前时间 < 活动开始时间 + 活动时长
- **已结束**：当前时间 ≥ 活动开始时间 + 活动时长

#### 活动报告 Banner
在基本信息卡片下方，以**独立的 Banner 按钮**形式展示活动报告入口：

##### 视觉风格
- 横向通栏卡片，圆角，带有轻微的渐变背景（如浅琥珀色到白色）
- 左侧：文档/报告图标 + 「活动报告」标题
- 右侧：状态文案 + 箭头图标

##### 状态展示
| 状态 | 右侧文案 | 背景风格 |
|------|----------|----------|
| 已生成 | 「查看报告 →」 | 可点击，带 hover 反馈 |
| 未生成 | 「报告生成中…」 | 置灰，不可点击 |

##### 交互
- **已生成**：点击 Banner → 打开活动级别的 H5 报告页面
- **未生成**：点击无跳转，仅展示提示「报告未生成完毕，请稍后查看」
- 实现阶段可先用空白页 + 文字描述临时标记该 H5 页面

#### 参与教师
- 标题栏：左侧琥珀色竖条 + 「参与教师」+ 人数统计
- 内容区：白色卡片内横向平铺教师姓名标签

#### 参与学生
- 标题栏：左侧绿色竖条 + 「参与学生」+ 人数统计
- 内容区：纵向排列的学生卡片列表

##### 学生卡片
每张卡片横向布局，包含以下内容：

| 元素 | 说明 |
|------|------|
| 幼儿头像 | 左侧圆形头像占位图，未上传时展示默认头像 |
| 幼儿姓名 | 如：王小明 |
| 报告状态 | 右侧展示该幼儿的个人报告生成状态 |

##### 报告状态说明
| 状态 | 展示文案 | 交互 |
|------|----------|------|
| 已生成 | 「查看」 | 点击跳转至该幼儿的 H5 个人报告页面 |
| 未生成 | 「生成中…」 | 不可点击，提示「报告未生成完毕，请稍后查看」 |

##### 学生卡片交互
- 点击整张卡片（仅限报告已生成时）→ 打开对应幼儿的 H5 个人报告页面
- 该报告页面与上方的「活动报告」为不同页面，实现阶段同样可先用空白页 + 文字描述临时标记
